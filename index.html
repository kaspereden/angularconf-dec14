Quick recap MVC is a circle. MVC IMAGE This works, the design pattern is already being used for years. We’re missing 2-way data binding. EXPLAIN WITH IMAGE MVC with two way databinding is called MVVM Model View ViewModel, I prefer to refer to it as MVC with 2-way databinding But what is 2 way databinding and why is it so cool? Lets go back to the previous example EXAMPLE 1 here we had to write all kind of things in order to make the model update the view and vice versa. This traditional MVC is using 1 way databinding. Data goes from the model -> view -> controller -> model 2 way binding goes either way. view <-> model This means that we can do some cool stuff like ‘$scope.value = “my first value”; and in the view we create {{value}} whenever either one of these changes they all change.
